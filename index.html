YouMeJS

The simple library ♥ helping developers creating powerful web widgets.

So... What is it?

YouMeJS is a Knockout inspired library, designed for web widget developers, also known as "People who make software for other people who make software".

Features

  • Knockout inspired template syntax
  • Seamless refresh
  • Events
  • Customizable and extensible library
  • Developer friendly

Requirements and compatibility.

YouMeJS requires jQuery on the client side and as yet to be tested on different browsers. If this interactive guide works for you, then YouMeJS is probably compatible with your browser.

Where to get it?

For now YouMe is only available at my Github repository under the /build/release folder. Get it here.

Getting started

As a web widget developer, your job is to provide another developer with a simple yet powerful API for him to use on his website. In this guide we will develop a Javascript object named MyAwesomeService which will act as such API. Lets get started!

// /js/mas.js
MyAwesomeService = {};

That's it! You are done. Now the client can load your code and get access to MyAwesomeService object.

<!-- /index.html -->
<script src="js/mas.js"></script>
<script>
console.log(MyAwesomeService);
</script>

But our newly created API doesn't do much, lets fix that with the YouMe() function.

// /js/mas.js
// We expose a application interface in the global space.
MyAwesomeService = {
  
  // The load function will our client's entry point.
  load: function()
  {
    // For the sake of the demo we're using jQuery to load YouMe,
    // but in a real case scenario youme.x.x.x.min.js and mas.js would be one file.
    var YouMePath = 'js/youme.0.0.4.min.js';
    $.getScript(YouMePath, function() {

      // We transform our application interface in an advanced application interface using YouMe
      MyAwesomeService = YouMe();

      // Then we expose a start() function to our client.
      // For the sake of the demo the parameter provided is a debug flag... But it doesn't have to be exposed in a real case..
      // We also provide a selector to work only on the demo part of the document... Think about the possibilities.
      MyAwesomeService.start = function(debug, selector) {

        // Finally we call fuse() (a function from the initial YouMe object).
        // The first argument is a selector.
        // The second argument is an name to use in document parsing. Here it's "mas".
        // The third parameter is a set of options.
        this.fuse(selector, 'mas', { // "mas" stands for "MyAwesomeService"... Obviously.
            debug: debug
        });
      };

    });
  }
};

It's a lot of comments but it's only 7 lines of code... Really.

Expose and display variables

As web widget developer, keep in mind that YouMe will take care of exposing a comprehensive API to your client, but it's you who provides its backend.
To do so you must provide the YouMe() function with a Storage object as a parameter. Like so:

// /js/mas.js

// [...]
var myStorage = /* Put here your code to create your own storage object */;
MyAwesomeService = YouMe(myStorage);
// [...]

Here is an example of a minimal Storage object:

// A constructor, feel free to add parameters such as API key, configuration or whatever.
var MyStorage = function()
{
  // Insert your own code here
};

// Set is called when YouMe tries to store a new variable in this storage.
MyStorage.prototype.set = function(name, value)
{
  // Insert your own code here

  return this; // Return "this" to allow chaining.
};

// Get is called when YouMe tries to retrieve a variable from this storage.
MyStorage.prototype.get = function(name, defaultValue)
{
  // Insert your own code here

  return defaultValue; // If your storage doesn't contain the requested variable, please return the defaultValue argument.
};

// Unset is called when YouMe tries to erase a variable from this storage.
MyStorage.prototype.unset = function(key)
{
  // Insert your own code here

  return this; // Return "this" to allow chaining.
};

// Has is called when YouMe checks if a variable exists in this storage.
MyStorage.prototype.has = function(key)
{
  // Insert your own code here

  return false; // Return false if you don't "has" it. :3
};

// Save is called whenever YouMe ask for storage to be persisted.
// It's usually when you send variables to your server.
MyStorage.prototype.save = function()
{
  // Insert your own code here

  return this; // Return "this" to allow chaining.
};

As you can see a Storage object must implement at least set(), unset(), has(), get(), and save(). You can include any logic behind those functions though.

Conditions

Without requiring you to change any line in your Storage object, YouMe offers the possibility for your client to declare conditions in his HTML page.
Here are some examples:

<p data-mas="if: aSimpleTrueBoolean">This paragraph should be shown</p>
<!-- mas if: aSimpleTrueBoolean -->
<p>This paragraph will be shown because aSimpleTrueBoolean is evaluated as true.</p>
<!-- /mas -->

<p data-mas="if: aSimpleFalseBoolean">But not this one.</p>
<!-- mas if: aSimpleFalseBoolean -->
<p>This paragraph will be hidden because aSimpleFalseBoolean is evaluated as false.</p>
<!-- /mas -->

<p data-mas="if: aSimpleTrueBoolean && aSimpleFalseBoolean">This paragraph will be hidden, thanks to our simple logic evaluation.</p>
<p data-mas="if: aSimpleTrueBoolean || aSimpleFalseBoolean">This paragraph will be visible, thanks to our simple logic evaluation.</p>

<p><a href="#" data-mas='attribute: {"title": "I am a tooltip!"} {"title": "aSimpleArray.length > 3"}'>This link will have tooltip</a></p>
<p><a href="#" data-mas='attribute: {"title": "I am a tooltip!"} {"title": "aSimpleArray.length == 3"}'>This link will not have a tooltip</a></p>

For the sake of brevity, we did not repeat the alternative comment syntax, but keep in mind that they are available.
For the same reason we did not show all operators, but know that ==, != > < >= <= || and && are all available.
Conditional attributes are a little bit tricky. Think of them as two JSON objects separated by a space: one for attributes, the other one for conditions.

Loops

YouMe also exposes a simple loop system for array variables. As usual some examples:

<!-- mas for: aSimpleArray -->
<p data-mas="text: context.item"></p>
<!-- /mas -->

<ul data-mas="for: aSimpleArray">
  <li>Index: <span data-mas="text: context.loopIndex"></span> / Length: <span data-mas="text: context.loopLength"></span></li>
</ul>

And the results

  • Index: / Length:

When using a loop a special variable get exposed: context.
The context variable exposes context.item, context.loopIndex and context.loopLength properties.

Set variables and save

YouMe provides ways for clients to set some of your Storage object variable too. Here is how a developer would set this up in his HTML page.

<form methd="get" action="">
  <input name="aSimpleVariable" type="text" value="" data-mas="input: aSimpleVariable" />
  <input type="button" data-mas="save" value="Save" />
</form>

And the result

What input does is just calling Storage.set() function of the Storage object. As for save it just calls Storage.save().
Even though we used a proper form, input can be added to any HTML control, as for save it can be added to any clickable HTML element.

Expose events and actions

Lets stop with HTML and move back to Javascript. YouMe exposes events for your client to listen to. Here are some Javascript examples.

MyAwesomeService.on('start', function () {
  console.log("This javascript listener is triggered on application's start (but before any refresh)") ;
});

MyAwesomeService.on('beforeRefresh', function () {
  console.log("This javascript listener is triggered just before refreshing") ;
});
MyAwesomeService.on('afterRefresh', function () {
  console.log("This javascript listener is triggered just after refreshing") ;
});

MyAwesomeService.on('beforeSave', function () {
  console.log("This javascript listener is triggered just before saving") ;
});
MyAwesomeService.on('afterSave', function () {
  console.log("This javascript listener is triggered just after saving") ;
});

These events are exposed through your API, but can only be accessed after YouMe() is called. To achieve this you might want to tweak the load() function that we defined earlier. You can check the source code of this page for a real example.

In Javascript nothing is really private. As such a client is free to call your Storage object functions whenever he wants.
And YouMe is making it even easier for them as your interface will expose the same set(), unset(), has(), get(), and save() functions as your storage but will additionally trigger a seamless refresh of the template.

MyAwesomeService.set("someVariable", "Some value");
MyAwesomeService.get("aSimpleVariable", "A default value");
MyAwesomeService.has("aSimpleVariable");
MyAwesomeService.unset("aVariable");
MyAwesomeService.save();

Calling those instead of the storage's one will also trigger events as an added bonus.

Extending YouMe

As a final feature YouMe exposes a simple extension mechanism. You can create your own template features with the same tools we used to create the built-in ones. Here is an example:

// /js/mas.js (After calling YouMe())
// Here we add a custom command
MyAwesomeService.addCommand('sum', function (command) {
  // We listen to the click event
  command.target.on('click', function () {
    // We do the operation using to command interface provided by YouMe
    var sum = parseInt(command.getArgument(0)) + parseInt(command.getArgument(1));

    // We set the HTML of the target HTML element
    command.target.setHtml(sum);
  });
});

And here is how your client will use it.

<p data-mas="sum: 1 3">Click to see sum in action!</p>
